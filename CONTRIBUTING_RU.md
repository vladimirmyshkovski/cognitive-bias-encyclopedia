# Вклад в энциклопедию когнитивных искажений

Спасибо за интерес к проекту! Это руководство для **людей**, которые хотят добавить новое искажение, улучшить существующее или перевести на другой язык. (Для AI-агентов есть отдельный [[AGENTS]].)

## Как внести вклад

### 1. Сначала откройте Issue

Перед началом работы откройте Issue с описанием:
- **Что** вы хотите добавить или изменить
- **Почему** это важно (ссылка на исследование, пример из жизни)
- **Какие источники** использовали

Это помогает избежать дублирования работы и согласовать подход.

### 2. Сделайте Fork и Clone

```
git clone https://github.com/YOUR_USERNAME/cognitive-bias-encyclopedia.git
cd cognitive-bias-encyclopedia
```

### 3. Создайте ветку

```
git checkout -b add-bias-name
# или
git checkout -b improve-existing-bias
# или
git checkout -b translate-to-language
```

### 4. Добавьте или измените запись

**Если добавляете новое искажение** — следуйте [[AGENTS]] (10 секций, English-first имена файлов, 6 категорий, 3+5 перекрёстных ссылок).

**Если улучшаете существующее** — добавьте примеры, источники, философские параллели. Не меняйте структуру без необходимости.

**Если переводите** — создайте `en/` (или другой `lang/`) директорию и продублируйте структуру `ru/`.

### 5. Проверьте качество

Прежде чем делать Pull Request, проверьте:

- [ ] Файл следует шаблону (10 секций в правильном порядке)
- [ ] Все wiki-links резолвятся (в Obsidian откройте файл и проверьте, что ссылки кликабельны)
- [ ] Категория правильная (одна из 6: суждение, память, само, социальное, внимание, вероятность)
- [ ] Минимум 2 академических источника + ссылка на Wikipedia
- [ ] Добавлен в [[4. Таксономия]] (если это новое искажение)
- [ ] Файл добавлен в таблицу в README.md

### 6. Pull Request

Создайте PR с описанием:
- **Что** изменено
- **Зачем** (Issue-ссылка)
- **Как проверить** (любые специальные инструкции)

Проверьте, что `obsidian tags --strict` возвращает 0 ошибок (если вы добавили новые теги).

---

## Чего НЕ делать

- ❌ **Не создавайте YAML frontmatter** — этот проект использует inline-метаданные
- ❌ **Не используйте русские имена файлов** — `Confirmation Bias.md`, не `Предвзятость подтверждения.md`
- ❌ **Не добавляйте секции вне шаблона** без согласования (создайте Issue сначала)
- ❌ **Не пропускайте `## Дialectical mirror`** — это уникальная особенность нашей энциклопедии
- ❌ **Не пишите от первого лица** — статьи должны быть в безличной форме

---

## Качество источников

Принимаются:
- Рецензируемые статьи (peer-reviewed journals)
- Книги от академических издательств
- Проверенные онлайн-ресурсы (Wikipedia, Britannica, YourBias.is)

**Не принимаются:**
- Блоги без авторства
- Reddit, Quora, Stack Exchange
- ИИ-сгенерированные тексты без редактирования человеком
- Самоцитирование (если только это не оригинальное исследование)

## Категории искажений

Каждая запись должна иметь **ровно одну** категорию:

| Категория | Когда использовать |
|-----------|--------------------|
| **суждение** | Оценка информации, аргументов, вариантов |
| **память** | Искажение воспоминаний |
| **само** | Самооценка, самоатрибуция |
| **социальное** | Влияние других людей, групп, иерархий |
| **внимание** | Что замечаем, что игнорируем |
| **вероятность** | Оценка вероятностей, рисков, шансов |

Полный список существующих искажений — в [[4. Таксономия]].
