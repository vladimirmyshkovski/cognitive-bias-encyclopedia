# AGENTS.md — Энциклопедия когнитивных искажений

> **Аудитория**: AI-агенты (Claude / GPT / другие), пишущие или редактирующие записи в `Энциклопедия когнитивных искажений/`.
> **Дата**: 2026-07-05
> **Версия**: 0.2
> **Главное правило**: **строгое соответствие шаблону**. Записи используются для сайта на Astro — каждое отклонение ломает вёрстку.

---

## 0. TL;DR

- Энциклопедия состоит из **фундаментальных вводных файлов** (0–3) и **каталога** (5. Каталог/) с отдельным файлом на каждое искажение.
- Каждая запись искажения = **10 секций в строгом порядке** + стандартный header.
- Шаблон-образец: **[[ru/5. Каталог/Confirmation Bias|Confirmation Bias.md]]** — копируй его структуру.
- Любая запись должна быть **самодостаточной** для чтения вне Obsidian (для Astro).

---

## 1. Что это за директория

Это **открытая русскоязычная энциклопедия когнитивных искажений**. Используется как:

1. **Личная база знаний** в Obsidian (с парными `[[wiki-ссылками]]`).
2. **Источник контента для сайта на Astro** — каждая запись рендерится как отдельная страница.

Из этого следует главное требование: **структура предсказуема**, потому что Astro читает frontmatter и секции программно.

**Где создавать новую запись**: только в `5. Каталог/`. Имя файла = имя искажения на английском + `.md`.

## 3. Канонический шаблон записи

**Образец для копирования**: `5. Каталог/Confirmation Bias.md`. Не отступай от него.

### 3.1. Header (обязательные метаданные)

```markdown
**Status:** `reference`
**Date:** YYYY-MM-DD
**Domain:** `encyclopedia`

# English Name (Русское название)
```

- `Status` всегда `reference` (это справочник, не черновик)
- `Date` — дата создания записи в формате ISO
- `Domain` всегда `encyclopedia`
- Теги **не нужны** в header записи (они уже в README и AGENTS)

### 3.2. Metadata block (сразу после заголовка)

```markdown
**Также известно как:** alias1, alias2, alias3
**Категория:** [категория] (краткое пояснение в скобках)
**Связанные:** [[ru/5. Каталог/Anchoring]], [[ru/5. Каталог/Confirmation Bias]], ...
```

- **Aliases**: 1–4 через запятую. Если нет — пиши `нет`
- **Категория**: одна из 6 канонических (см. §5)
- **Связанные**: 3–6 ссылок на другие записи в каталоге. Если искажение изолированное — добавь [[ru/5. Каталог/Motivated Reasoning]] как fallback

### 3.3. Separator

После metadata блока — одна строка `---`, затем пустая строка.

### 3.4. Десять обязательных секций в строгом порядке

| # | Секция | Что внутри | Объём |
|---|--------|------------|-------|
| 1 | `## Определение` | 1–2 абзаца, что это, чем отличается от смежных | 100–200 слов |
| 2 | `## Пример` | 3–5 подкатегорий: Бытовой, Политический, Научный (где релевантно), Исторический (если есть классическое исследование) | 200–400 слов |
| 3 | `## Механизм` | Нумерованный список 2–5 механизмов, объясняющих, **почему** мозг так делает | 100–250 слов |
| 4 | `## Эволюционная функция` | Зачем это искажение было полезно в саванне, почему вредно сейчас | 100–200 слов |
| 5 | `## Дialectical mirror` | Связь с [[Диалектическое и метафизическое мышление]]: тезис ↔ антитезис ↔ синтез | 80–150 слов |
| 6 | `## Философские параллели` | Стоики, Буддизм, Кант, Ницше, Сократ + ссылка на [[Доказательство отсутствия Бога как создателя]] (если уместно) | 150–250 слов |
| 7 | `## Как поймать` | 3–5 вопросов-индикаторов «когда ты ловишь себя» | 80–150 слов |
| 8 | `## Как смягчить` | Нумерованный список 3–7 конкретных техник | 100–200 слов |
| 9 | `## Связанные искажения` | Bullet-список `[[link]] — краткое пояснение` | 4–6 связей |
| 10 | `## Источники` | 3–5 академических источников + ссылка на Wikipedia + (опционально) YourBias.is | 4–7 строк |

**Не добавляй секции**, которых нет в этом списке, без явного запроса пользователя.
**Не пропускай секции**, даже если кажется, что искажение не имеет, например, эволюционной функции — напиши, что она неясна.

## 4. Naming Conventions

### 4.1. Имена файлов

- **English name + `.md`**: `Confirmation Bias.md`, `Halo Effect.md`
- **Без спецсимволов**: `Hindsight Bias.md`, **не** `Hindsight-Bias.md`
- **Без русского в имени файла**: имя файла всегда на английском
- **Без сокращений**: `Sunk Cost Fallacy.md`, **не** `SCF.md`

### 4.2. Заголовок H1

```markdown
# English Name (Русское название)
```

| Пример | Файл |
|--------|------|
| `Confirmation Bias` | `Confirmation Bias.md` |
| `Halo Effect` | `Halo Effect.md` |
| `Loss Aversion` | `Loss Aversion.md` |
| `Motivated Reasoning` | `Motivated Reasoning.md` |
| `Just-World Fallacy` | `Just-World Fallacy.md` |

Если устоявшегося русского перевода нет — пиши только английское:
- `Curse of Knowledge` (без перевода)
- `Pygmalion Effect` (без перевода)

### 4.3. Кавычки в русском

Используй «ёлочки» (« »), не "лапки". Это канон русского академического стиля.

## 5. Шесть канонических категорий

Каждая запись должна иметь **ровно одну** категорию из списка:

| Категория | Когда использовать | Примеры в каталоге |
|-----------|--------------------|--------------------|
| **суждение** | Оценка информации, аргументов, вариантов | Anchoring, Confirmation Bias, Loss Aversion |
| **память** | Искажение воспоминаний, hindsight | Hindsight Bias |
| **само** | Самооценка, самоатрибуция | Dunning-Kruger, Self-serving Bias, Illusion of Control |
| **социальное** | Влияние других людей, групп, иерархии | In-group Bias, Halo Effect, Authority Bias |
| **внимание** | Что замечаем, что игнорируем | Availability Heuristic, Survivorship Bias |
| **вероятность** | Оценка вероятностей, рисков, шансов | Gambler's Fallacy, Base Rate Fallacy, Planning Fallacy |

Если искажение плохо ложится в одну категорию — **выбери главную**, остальные упомяни в `## Механизм`.

## 6. Связи (wiki-links)

### 6.1. Обязательные ссылки

Каждая запись **обязательно** должна ссылаться на:

1. **[[Диалектическое и метафизическое мышление]]** — в секции `## Дialectical mirror`
2. **3–6 других искажений** — в `## Связанные искажения`
3. **Wikipedia** — в `## Источники`

### 6.2. Формат ссылок

```markdown
[[Имя файла без .md]]                    ← просто ссылка
[[Имя файла без .md|Отображаемый текст]]  ← с алиасом
```

**Не** используй URL внутри `[[]]` — только внутренние ссылки Obsidian.

### 6.3. Антиизоляция

Запись **не должна быть изолированной** в графе. Если ты создаёшь новое искажение, проверь:

- [ ] Минимум 3 перекрёстных ссылки в `## Связанные искажения`
- [ ] Хотя бы одна запись в энциклопедии ссылается **на новую** в `## Связанные искажения` (обнови 1–2 существующих записи, если нужно)
- [ ] Запись включена в таблицу в `README.md` (обнови таблицу)

## 8. Источники

### 8.1. Формат

```markdown
- Author, A. B. (Year). *Title of the work*. Publisher / Journal, vol(issue), pages.
- [Wikipedia: Article name](https://en.wikipedia.org/wiki/Article_name)
```

### 8.2. Требования

- **Минимум 2 академических источника** (книга или рецензируемая статья)
- **Минимум 1 ссылка на Wikipedia** — для проверки и как быстрый линк для читателя
- **Опционально**: ссылка на [YourBias.is](https://yourbias.is) для коротких, человекочитаемых объяснений

### 8.3. Что НЕ считается источником

- Блоги без авторства
- Reddit, Quora, Stack Exchange
- Самоцитирование (если только это не оригинальное исследование)
- ИИ-сгенерированные тексты

## 9. Чек-лист перед добавлением нового искажения

Прежде чем создавать файл, ответь:

- [ ] **Существующее покрытие**: этого искажения нет в каталоге? (проверь через grep по `5. Каталог/`)
- [ ] **Значимость**: искажение признано в академической литературе (есть публикации, оно есть в Wikipedia, оно не выдумано)?
- [ ] **Релевантность читателю**: оно полезно для целевой аудитории энциклопедии (русскоязычные читатели, интересующиеся саморазвитием)?
- [ ] **Источники**: я могу найти минимум 2 академических источника?

Если хотя бы один пункт «нет» — **не создавай запись**. Объясни пользователю, почему не стоит.

### Чек-лист самой записи (перед коммитом):

- [ ] Header: `Status: reference`, `Date: YYYY-MM-DD`, `Domain: encyclopedia`
- [ ] H1: `# English Name (Русское название)`
- [ ] Metadata block: Также известно / Категория / Связанные
- [ ] Все 10 секций в правильном порядке
- [ ] Хотя бы одна ссылка на [[Диалектическое и метафизическое мышление]]
- [ ] 3–6 перекрёстных ссылок в `## Связанные искажения`
- [ ] Обновил `README.md` (добавил в таблицу)
- [ ] Обновил 1–2 существующих записи (добавил ссылку на новое искажение в их `## Связанные искажения`)
- [ ] Запустил `obsidian tags --strict` — 0 ошибок
- [ ] Если категория новая — добавил в §5 этого файла

## 10. Anti-examples (что НЕ делать)

### ❌ Неправильно: неполная запись

```markdown
# Confirmation Bias

Confirmation bias — это когда мы ищем информацию, подтверждающую наши убеждения.

## Пример
Курильщик читает статью о вреде курения и помнит только одно исследование.

## Источники
- Wikipedia
```

**Почему плохо**: нет 9 из 10 секций. Astro не сможет отрендерить предсказуемо. Нет cross-links. Один источник.

### ❌ Неправильно: длинно и без структуры

```markdown
# Dunning-Kruger

Это очень интересное искажение, которое было открыто в 1999 году двумя исследователями...
[2000 слов эссе без подзаголовков, без примеров, без механизма]
```

**Почему плохо**: Astro не знает, что с этим делать. Нет нумерованных списков. Нельзя быстро найти «Как смягчить».

### ❌ Неправильно: русский заголовок

```markdown
# Предвзятость подтверждения (Confirmation Bias)
```

**Почему плохо**: имя файла и H1 должны быть **English-first**. Файл — `Confirmation Bias.md`, H1 — `# Confirmation Bias (...)`.

### ❌ Неправильно: YAML frontmatter вместо inline

```markdown
---
title: Confirmation Bias
status: reference
tags: [когнитивные_искажения]
---

# Confirmation Bias
```

**Почему плохо**: этот vault использует **inline-метаданные** (`**Status:** \`reference\``), а не YAML frontmatter. Astro читает именно inline. См. корневой `[[AGENTS]]`.

### ❌ Неправильно: нет ссылок на другие искажения

```markdown
## Связанные искажения
Ничего не приходит в голову.
```

**Почему плохо**: запись изолирована. Этот vault — **граф знаний**. Каждая запись должна иметь 3–6 перекрёстных ссылок.

### ❌ Неправильно: «нет» в категории или aliases

```markdown
**Также известно как:** нет
**Категория:** ?
```

**Почему плохо**: категория — одна из 6 канонических (см. §5). Aliases — либо 1–4 синонима, либо убери строку.

## 11. Workflow: добавление нового искажения

```
1. Проверить, что искажения нет в каталоге
   ↓
2. Найти 2+ академических источника
   ↓
3. Создать файл `5. Каталог/<English Name>.md`
   ↓
4. Скопировать шаблон из Confirmation Bias.md
   ↓
5. Заполнить все 10 секций
   ↓
6. Проверить чек-лист из §9
   ↓
7. Обновить README.md (добавить в таблицу, обновить дерево структуры)
   ↓
8. Обновить 1–2 существующих записи (cross-link на новое)
   ↓
9. Запустить `~/.local/bin/obsidian tags --strict`
   ↓
10. Готово
```

## 13. Связь с Astro-сайтом

Каждая запись каталога рендерится как **отдельная страница** на сайте. Astro ожидает:

- Предсказуемый header (3 inline-поля)
- H1 в формате `# English (Русский)`
- 10 секций в фиксированном порядке
- Wikipedia-ссылку в источниках (для meta-тегов og:image и т.п.)

**Любое отклонение от шаблона = сломанный рендеринг**.
